As três fases - resumo

MVP Piloto

Objetivo: validar processo com fornecedores reais.

  • Formulário com login por ID
  • Dashboard com usuário/senha
  • Supabase + FUP no Storage
  • Streamlit Cloud
  • Secrets fora do Git

Prazo: já em andamento

Básica Oficializar

Objetivo: uso formal com pouco esforço extra.

  • Domínio fornecedores.alcoa.com.br
  • IDs estáveis (tabela fornecedores)
  • Senha dashboard em hash
  • Backup automático Supabase
  • RLS revisada

Prazo: 2–4 semanas

Forte Corporativo

Objetivo: padrão TI Alcoa / compliance.

  • Azure Container Apps / App Service
  • SSO (Azure AD) no dashboard
  • VPN - dashboard só intranet
  • FUP via API / SAP
  • Logs, alertas, auditoria

Prazo: 1–3 meses+

Segurança e observabilidade - comparativo

Aspecto MVP Produção básica Produção forte
Acesso fornecedor ID numérico (código) ID fixo no banco E-mail + token ou código SAP
Acesso equipe Usuário + senha (Secrets) Senha em hash, vários usuários SSO Azure AD + VPN
Dados sensíveis Fora do Git, Storage Supabase + backup diário Key Vault, rede interna
Banco Supabase free/pro, RLS aberta RLS revisada, índice único PostgreSQL corporativo
Rastreio envios codigo_fornecedor + timestamps + IDs estáveis + auditoria completa
Visualização Dashboard + export Excel + domínio corporativo + Power BI / logs Splunk
Infraestrutura Streamlit Cloud Docker ou Cloud + CNAME Azure aprovado pela TI
Quem aprova Equipe do projeto Gestor + compras TI + segurança + compliance
